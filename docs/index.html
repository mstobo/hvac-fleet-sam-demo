Industrial telemetry at scale — without burning tokens

High-volume MQTT is not an LLM input stream. This reference demo is built on a real-time, event-driven pipeline on Solace — each sensor reading is a broker event, and every stage reacts in flow, not in batch. Generative models sit behind event-triggered reasoning as well as operator chat, so AI runs when the mesh signals value, not on a schedule.

The problem

Most IIoT + AI pilots fail at scale because cost and latency grow linearly with sensor volume. The fix is architectural: deterministic processing first, agents second.

Value framework

1 · Protect spend

Keep the LLM off the firehose

Every unchanged temperature reading does not need a model call. Budget and latency stay predictable as fleet size grows.

2 · Signal over noise

Filter in the event stream

Deadband and heartbeat logic run on live MQTT events — suppressing noise and forwarding only meaningful change as it happens, not after a batch job.

3 · Sketch, don’t spell out

Compact summaries at publish time

Template natural-language sketches turn telemetry into operator-ready signal without generative cost — aligned with the sketch-not-every-token pattern in Sketch-of-Thought (SoT) research.

4 · Rules before models

Detect on events, not inference

Zones, spikes, and fleet-critical fractions emit alerts and MQTT triggers deterministically as events arrive — sub-second operator signal.

5 · AI where ROI is clear

Event-triggered reasoning + chat

SAM Fleet Query answers in the Web UI, but fleet-wide automation also fires when the mesh publishes a high-value trigger — reasoning driven by events, not chat alone.

Two planes, one broker

Data plane — microservices subscribe and publish on MQTT topics in real time: filter → sketch → detect → store. No LLM on the hot path; latency stays bounded by broker delivery and deterministic code.

Control plane — Solace Agent Mesh gateways react to curated state and mesh events: operator questions over HTTP/SSE, plus automated fleet analysis when rules declare fleet critical and publish an analysis request.

What this demo proves

A live HVAC-style fleet on Solace PubSub+ — event-driven pipeline, ~99% MQTT noise reduction, template sketches, rule-based detection, and SAM analysis triggered by operators or mesh events. The live dashboard on AWS shows the stream in motion; these pages explain why it matters.
