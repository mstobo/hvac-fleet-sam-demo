The live demo on AWS runs an event-driven MQTT pipeline in real time, with SAM reasoning on operator chat and on mesh events (e.g. fleet-critical triggers) — not batch analytics and not chat-only AI. Service names, topics, and deployment steps live in the repository README.

How it flows (real-time, event-driven)

Ingest

Sensors publish MQTT events to Solace continuously — the mesh is the nervous system.

Filter

Each event is evaluated in stream; noise is dropped before the next stage sees it.

Sketch

Deterministic NL summaries on forwarded events—operator language without an LLM per sample. Related idea: Sketch-of-Thought (reasoning compression, not this pipeline).

Detect

Rules fire on live events; fleet-critical conditions publish triggers to the mesh.

Act (events + chat)

Dashboards and Slack on alerts; SAM chat for questions; automated fleet analysis when the mesh requests it.
